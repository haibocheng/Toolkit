trackOmniture()method 
public function trackOmniture(trackID:String, doTrack:Boolean = true, reset:Boolean = false, initObject:Object = null, delay:Number = 0):void

The main method to call when tracking Omniture using XML

Example:

  • TrackingManager.getInstance().init( trackingXML, _level0 );
  • TrackingManager.trackOmniture( "home" );
  • TrackingManager.trackOmniture( "products", false, true );

Parameters

trackID:String — The item ID specified in the XML to track.
 
doTrack:Boolean (default = true) — A flag indicating if the ActionSource track() method should be invoked. This is useful should you want to set the internal object variables but not actually track anything.
 
reset:Boolean (default = false) — A flag indicating if the internal data object should be reset. If not specified the internal object will retain the previous values set and send those when the ActionSource track() method is invoked.
 
initObject:Object (default = null) — Specify this object should you want to manually override variables. Object should be in the format obj = {key: value}; or obj = {prop2: "hello world"}
 
delay:Number (default = 0) — A delay in MS for when the ActionSource track() method is invoked.
